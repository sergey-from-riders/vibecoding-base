# ADR-0002: DB ID naming contract (`<entity>_id`, UUID-only)

- Status: accepted
- Date: 2026-03-01

## Context
Для многокомандной и агентной разработки нужна абсолютно однозначная схема именования ключей в БД.
Без этого растет количество ошибок в миграциях, join-запросах и контрактных моделях.

## Decision
Принят жесткий контракт:
- Entity table PK: строго `<entity>_id`.
- Тип PK/FK доменных сущностей: строго `uuid`.
- FK имя: строго `<target_entity>_id`.
- Generic имена (`id`, `uuid`, `entity_id`) для доменных ключей запрещены.

Формальные правила закреплены в `docs/07-DB-NAMING-CONVENTIONS.md`.

## Consequences
- Единообразные SQL-модели и API DTO.
- Проще автогенерация контрактов и SDK.
- Меньше неоднозначности при работе агентов.
- Любая миграция с нарушением правила не принимается.
