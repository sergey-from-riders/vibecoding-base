# tools/scripts

Developer scripts for contract checks, migrations, and deployment helpers.

## Scripts
- `check_starter_integrity.sh` — offline baseline for the starter template
  - validates mandatory starter files
  - runs DB contract and empty app limit checks
  - usage:
    - `tools/scripts/check_starter_integrity.sh`
- `check_db_contract.sh` — static DB contract check for migration files
  - verifies required auth/company/session/device tables and version tables
  - verifies paired up/down migrations
  - fails on generic `id` or serial domain id patterns
  - usage:
    - `tools/scripts/check_db_contract.sh`
- `deploy_remote.sh` — image-based deployment to remote server using Docker Compose
  - syncs deployment manifests (`infra/docker`, `infra/nginx`)
  - pulls prebuilt container images from registry on server
  - expects:
    - `DEPLOY_HOST` (e.g. `user@server`)
    - `DEPLOY_PATH` (e.g. `/opt/ai-app`)
- `check_python_limits.sh` — validates strict Python backend numeric limits
  - file/function/handler/routes line limits
  - comment ratio limit
  - fails with non-zero exit code on any violation
  - usage:
    - `tools/scripts/check_python_limits.sh`
    - `tools/scripts/check_python_limits.sh apps/api`
- `check_web_limits.sh` — validates strict React/Node frontend numeric and style limits
  - ts/tsx file and function/component line limits
  - comment ratio limit
  - shadcn-only interactive primitives check
  - no-shadow / no-black-border / no-transition-all checks
  - disallow `fetch` outside shared api-client/server layers
  - token color count check (`<= 5`)
  - fails with non-zero exit code on any violation
  - usage:
    - `tools/scripts/check_web_limits.sh`
    - `tools/scripts/check_web_limits.sh apps/web`
- `check_cpp_limits.sh` — validates strict C++/Qt memory-safety and size limits
  - cpp/hpp file and function line limits
  - comment ratio limit
  - forbids `new/delete/malloc/free` in feature code
  - checks for `unique_ptr.release()` and `shared_ptr<QObject>` usage
  - fails with non-zero exit code on any violation
  - usage:
    - `tools/scripts/check_cpp_limits.sh`
    - `tools/scripts/check_cpp_limits.sh apps/desktop/qt`
- `check_template_clean.sh` — blocks old project residues and common secret shapes
  - fails on old branding, old env prefixes, old domains, private host/path fragments, and common key formats
  - usage:
    - `tools/scripts/check_template_clean.sh`
- `build_openapi_bundle.sh` — bundles modular OpenAPI into publishable artifacts
  - lint via Redocly
  - outputs:
    - `packages/contracts/openapi/dist/openapi.yaml`
    - `packages/contracts/openapi/dist/openapi.json`
    - `packages/contracts/openapi/dist/openapi.html` (best effort; may be skipped by current OpenAPI/tooling support)
    - `packages/contracts/openapi/dist/endpoints.catalog.json`
    - `packages/contracts/openapi/dist/postman.collection.json` (best effort)
  - usage:
    - `tools/scripts/build_openapi_bundle.sh`
- `check_openapi_coverage.sh` — validates endpoint coverage and examples
  - checks every endpoint from `endpoints.inventory.tsv` exists in OpenAPI
  - checks request examples, response examples, code samples, and mandatory response headers
  - usage:
    - `tools/scripts/check_openapi_coverage.sh`
