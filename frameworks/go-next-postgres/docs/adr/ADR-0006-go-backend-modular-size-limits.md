# ADR-0006: Strict modular Go backend with numeric size limits

- Status: accepted
- Date: 2026-03-02

## Context
Нужен backend-стандарт, который одинаково понимается и людьми, и агентами без двусмысленностей.
Главные боли: большие хендлеры, монолитные роутеры, сложное чтение кода в многокомандной разработке.

## Decision
Принят жесткий числовой стандарт:
- модульная структура `platform + modules`;
- маршруты разнесены по модульным файлам;
- endpoint = отдельный handler-файл;
- фиксированные лимиты строк/сложности/вложенности;
- минимальный процент комментариев;
- запрет бизнес-логики в handler;
- обязательные integration tests для каждого backend-модуля.

Канонический регламент:
- `docs/18-GO-BACKEND-ENGINEERING-STANDARD.md`

## Consequences
- Выше дисциплина и предсказуемость кодовой базы.
- Ниже скорость “быстрых хаотичных” правок.
- Выше читаемость и устойчивость при параллельной работе нескольких разработчиков.
