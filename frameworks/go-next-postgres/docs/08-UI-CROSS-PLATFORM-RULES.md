# 08. UI Cross-Platform Rules (Mandatory)

Этот документ — единый UI-контракт для Web, Telegram Mini App, Qt Desktop и Android (WebView-first).

## 1) Единая визуальная система
- Один глобальный акцентный цвет: `#0A84FF`.
- Акцент используется только для primary actions, активных состояний, выделений и ссылок.
- Запрещено вводить второй бренд-акцент без отдельного ADR.
- Максимальное число базовых цветов проекта: `5`.
- Тени как декоративный прием: запрещены (`0`).
- Черные рамки (`#000`) в UI: запрещены (`0`).

## 2) Token-first подход
- Источник токенов: `packages/tokens/tokens.json`.
- Любой UI-клиент обязан строить тему из токенов, а не из ad-hoc значений.
- Изменение цвета/типографики/spacing делается только через tokens + changelog.

## 3) UX-инварианты для всех платформ
- Информационная архитектура, названия сущностей, статусные состояния и ошибки должны быть одинаковыми.
- Модель форм и валидаций должна быть идентичной (по смыслу) во всех клиентах.
- Структура экранов auth/switch-company должна быть одинаковой по сценарию.

## 4) Accessibility (единый минимум)
- Цвет не единственный носитель состояния.
- Контраст и читаемость обязательны.
- Поддержка dynamic type / scaling, где платформа это предоставляет.
- Клавиатурная/assistive навигация обязательна для web/desktop.

## 5) Platform-native реализация, а не pixel-copy
- Мы унифицируем правила и токены, но не копируем 1:1 внешний вид между платформами.
- Для Android в текущем этапе используется WebView-first подход (web UI inside native shell).

## 6) Scope текущего этапа
UI-правила распространяются только на:
- auth
- switch active company

## 7) Telegram Web App совместимость (обязательно для web)
- Web-клиент должен корректно работать как обычный web и как Telegram Mini App.
- Тема должна уметь принимать `tgWebAppThemeParams`.
- Layout должен учитывать viewport/safe area в Telegram-контейнере.
- Нельзя ломать UX вне Telegram ради Telegram-специфики: нужен адаптерный слой.
