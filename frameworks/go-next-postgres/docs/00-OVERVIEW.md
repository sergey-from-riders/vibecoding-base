# 00. Обзор

## Название
`Go Next Postgres Framework`

## Режим разработки
AI-native/vibe coding 2026:
- архитектура и контракты описываются заранее;
- имплементация выполняется агентами короткими вертикальными срезами;
- тесты и observability обязательны в каждой итерации.

## Текущая граница шаблона
В baseline входят только два рабочих домена:
1. `Auth`
2. `Company Context (switch active company)`

## Почему так
Это минимальный мульти-тенант фундамент. Без него любые другие модули (orders/events/и т.д.) будут нестабильны и дороги в поддержке.

## Что взято как инженерный baseline
Шаблон не содержит host/path metadata исходных проектов, реальные секреты или приватные endpoint-ы.

В baseline оставлены только универсальные рабочие основы:
- service-layer backend;
- Next.js web adapter;
- PostgreSQL migration-first;
- модель active company для мультитенантности.

## Что изменено относительно референса
- монорепа проектируется сразу как polyglot (Go + TS + Qt);
- добавлен explicit layer для MCP/agents;
- контрактный и observability подходы сделаны обязательными с первого дня.
